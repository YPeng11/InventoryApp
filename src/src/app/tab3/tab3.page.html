Privacy and Security Requirements

This application interacts with a remote database (https://prog2005.it.scu.edu.au/ArtGalley) to manage inventory data. Please be aware of the following:

  • Data Transmission: Data submitted (item details) is sent over the network via HTTPS, providing encryption during transit. Ensure you are on a trusted network.
  • Data Storage: Inventory data is stored on the remote server. No inventory data is persistently stored locally on your device.
  • Authentication: This educational version may lack user authentication. Access control relies on the API's configuration. Real-world apps require robust security.
  • Unique Item Names: The backend enforces unique item names. Adding an item with a duplicate name will result in an error. Updates target items by their unique name.
  • Input Validation: The app validates input (required fields, numeric types) before submission to maintain data integrity.
  • Third-Party API: The application depends on the provided backend API. Its security and availability are managed externally.
  • No Personal Data Collection: The app is designed for inventory management and does not intentionally collect personal user data unless entered into inventory fields (e.g., 'Supplier Name').
  • Security Disclaimer: This is an educational tool. Security measures may be simplified. Avoid using sensitive, real-world data.

Use this application responsibly.
